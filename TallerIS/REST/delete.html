El método DELETE

Introducción

DELETE es el cuarto de los métodos HTTP que forman la base de las aplicaciones de gestión basadas en operaciones CRUD: Create, Retrieve, Update and Delete.

En este capítulo vamos a ver cuales son las características de este método y las facilidades que ofrece Jersey para definir un método que atienda peticiones DELETE.

Finalmente veremos la correspondencia entre los métodos HTTP vistos hasta este momento y las operaciones CRUD.

Contenidos

  1. El método DELETE.
  2. Aplicaciones con servicios CRUD.

El método DELETE

El método DELETE nos sirve para eliminar recursos del servidor.

El recurso a eliminar está identificado por la URI que recibe la petición.

El cuerpo de la petición está vacío.

El método DELETE es idempotente.

El método DELETE no es ni seguro ni cacheable.

El método DELETE

Los códigos de respuesta pueden ser:

  • 200 Ok. El cuerpo de la respuesta incluye una descripción de lo ocurrido.
  • 204 No content. Si el borrado se efectuó sin problemas. El cuerpo de la respuesta está vacío.
  • 202 Accepted. La petición está en curso. En esta caso en la cabecera Location de la respuesta se indica la URI para consultar el estado de la petición.
  • Si algo ha ido mal, el código de error correspondiente.

Normalmente, el código de respuesta es 204 No content para indicar que el recurso se ha eliminado sin problemas.

El método DELETE

De nuevo, Jersey pone a nuestra disposición una anotación para indicar que un método atenderá peticiones DELETE.

La anotación es @DELETE. En este caso, como ya conoces, la anotación @Path para extraer una parte de la URI que recibe la petición te puede ser de utilidad.

Aplicaciones con servicios CRUD

En este punto del curso ya hemos visto cuatro métodos del protocolo HTTP: GET, POST, PUT y DELETE.

Esto cuatro métodos se corresponden con las operaciones básicas de las aplicaciones de gestión CRUD: Create, Retrieve, Update and Delete.

La correspondencia que se suele establecer entre los métodos del protocolo HTTP y las operaciones CRUD es la siguiente:

  • Create → POST.
  • Retrieve → GET.
  • Update → PUT.
  • Delete → DELETE.

Aunque ya sabes que para crear un nuevo recurso en el servidor puedes utilizar tanto PUT como POST.

Resumen

Como en el caso de los métodos HTTP vistos anteriormente, Jersey ofrece una anotación para indicar que un método Java atenderá peticiones DELETE.

Los cuatro métodos vistos hasta este momento constituyen el corazón de las aplicaciones basadas en servicios CRUD.

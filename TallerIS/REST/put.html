El método PUT

Introducción

La semántica del método PUT es la de creación o actualización de recursos en el servidor.

Es muy semejante al método POST de hecho las operaciones son las misma, la diferencia estriba en el significado de la URI que recibe la petición.

Contenidos

  1. El método PUT.

El método PUT

El método PUT nos sirve para crear o actualizar recursos en el servidor.

La URI que recibe la petición representa la URI del recurso que se va a crear o actualizar.

El código de estado puede ser tanto 200 Ok como 204 No Content.

Si se devuelve el código 200 Ok, el cuerpo de la respuesta debe contener una representación del recurso creado.

Si se devuelve un código 204 No Content, el código de la repuesta estará vacío.

El método PUT

Las mismas técnicas que utilizamos con el método POST las podemos utilizar con el método PUT.

  1. Negociación de contenido con la anotación @Consumes.
  2. Lectura de parámetros de un formulario con la anotación @FormParam.
  3. Inyección y creación de URIs.

Resumen

Debemos utilizar el métodos PUT para crear o actulizar recursos en el servidor.

La diferencia con el método POST radica en el significado de la URI que recibe la petición: en el método PUT la URI es la del recurso que se va a modificar o crear.

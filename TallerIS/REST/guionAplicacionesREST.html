Guión para el desarrollo de aplicaciones REST

Introducción

Como hemos visto, no todas las aplicaciones web que se dicen diseñadas bajo los principios REST lo son realmente.

En este capítulo vamos a ver una clasificación de las aplicaciones web que, de manera progresiva, se van acercando a la arquitectura RESTful.

También veremos un guión que nos ayudará a construir nuestras aplicaciones web según la arquitectura REST.

Contenidos

Richardson matturity model

Muchas aplicaciones con el calificativo de REST realmente no lo son.

Para que nuestra aplicación pueda ser llamada REST (se dice que las aplicaciones son RESTful), han de hacer uso de los Conceptos y Propiedades que vimos en el capítulo anterior.

El modelo de madurez de Richardson clasifica a las aplicaciones por su grado de cumplimiento de los principios REST.

Richardson matturity model

Nivel 0

Los servicios de Nivel 0 utilizan HTTP como capa de transporte. Nada más.

Muchos de ellos exponen una sola URI, que es la puerta de entrada a la aplicación Web.

Muchas de las aplicaciones SOAP se encuentran en este nivel, por ejemplo Flickr.

Richardson matturity model

Nivel 1

Existen URIs para cada uno de los recursos que se exponen, pero no se sigue la semántica de los métodos HTTP.

Usualmente se utiliza GET para operaciones de lectura y POST para todo lo demás (crear recurso, actualizar recursos, eliminar recurso).

Richardson matturity model

Nivel 2

En este nivel, las aplicaciones exponen URIs para acceder a recursos. Y se hace el uso correcto de la semántica de los métodos.

Pero, no existen enlaces entre los recursos, dicho de otra manera, no se hace uso de la hipermedia.

Algunos ejemplos son el servicio S3 de Amazon.

Richardson matturity model

Nivel 3

Estas son las aplicaciones RESTful, cumplen con todos los principios REST:

  1. Exponen los recursos mediante URIs.
  2. Cada recurso define una o más representaciones.
  3. Usan los métodos HTTP con su semántica.
  4. Los recursos están enlazados, contienen liks a otros recursos.

Guía para la creación de aplicaciones REST

Crear aplicaciones RESTful es fácil, basta seguir los principios REST.

  1. Determina cuales son los recursos de tu aplicación.
  2. Determina cuales son las representaciones válidas para tus recursos.
  3. Define URIs para acceder a ellos.
  4. Define los métodos que pueden actuar sobre cada uno de los recursos a través de sus URIs.
  5. Determina los enlaces entre los distintos recursos e inclúyelos en sus representaciones.

Resumen

El modelo de madurez de Richardson nos permite clasificar una aplicación web por su grado de proximidad a la arquitectura REST.

Pero, además de esto, nos ha permitido encontrar un sencillo guión para poder construir aplicaciones web bajo esta arquitectura y que puedan ser llamadas, sin ambages, aplicaciones web RESTful.

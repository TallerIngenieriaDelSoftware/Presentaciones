BDD con Cucumber

Historias de usuario

Como ya has visto en la asignatura Paradigmas del Software (EI1048), las historias de usuario son un medio para facilitar que los usuarios especifiquen la funcionalidad que desean que tenga el software.

Generalmente se especifican con tripletas: As a role, I want goal/desire so that benefit.

Como usuario de la aplicación quiero que, al pulsar el botón Calcular se actualice la página web mostrando el resultado del cálculo.

Pruebas de aceptación

Una vez definida una historia de usuario, se debe acordar de qué modo se va a comprobar la nueva funcionalidad dando algún ejemplo concreto.

De nuevo, se una prueba de aceptación se especifica como una tripleta Given/When/Then.

Si la operación es una suma, y los número son dos y dos, el resultado debe ser cuatro.

Automatización con Cucumber

Lo que nos permite Cucumber es especificar una historia de usuario, y sus pruebas de aceptación asociadas, y generar a partir de ellas pruebas unitarias con JUnit.
